# ReAct Workflow: Facebook Surfer Agent

**Purpose:** Document how the agent reasons and acts through tasks

---

## What is ReAct?

**ReAct** = **Re**asoning + **Act**ing

The agent loops through:
1. **THOUGHT** - Reason about current state
2. **ACTION** - Call a tool
3. **OBSERVATION** - Get result
4. Repeat until task complete

## Key Design Decisions

| Decision | Rationale |
|----------|-----------|
| **CSS before Vision** | CSS is instant, vision takes ~5s and costs API credits |
| **Memory at start** | Reuse known patterns before planning |
| **Save on success** | Only store working patterns (filter failures) |
| **Semantic search** | Natural language queries match related tasks |
| **OpenAI embeddings** | Best quality for Facebook-specific terminology |
